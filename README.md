# 🧠 DENZ (덴즈)

> **DENZ는 사용자의 일상 기록을 안전하게 보관하고,  
> 누적된 데이터를 기반으로 개인화된 AI 인사이트를 제공하는 플래너 서비스입니다.**

---

## 📌 프로젝트 개요

- **서비스명**: DENZ
- **목적**:  
  일기 기록을 통해 사용자의 감정, 패턴, 방향성을 이해하고  
  AI 피드백으로 더 나은 내일을 설계하도록 돕는 것
- **핵심 원칙**
  - 프라이버시 우선 설계
  - 사용자 중심 AI 개인화
  - 단순하지만 확장 가능한 구조

---

## 🎯 MVP v1.0 범위

- 사용자는 일기를 작성하고 저장할 수 있다.
- 저장된 일기를 기반으로 AI 피드백을 정기적으로 제공받는다.
- 사용자는 원하는 시간과 기간 단위로 피드백을 확인할 수 있다.
- 모든 개인 일기 데이터는 **암호화되어 저장**된다.

---

## 🧩 주요 기능

### 1️⃣ 사용자 관리

- 회원 가입 / 로그인 / 로그아웃
- 사용자 정보 수정
- 회원 탈퇴

**보안**

- 비밀번호는 `bcrypt` 기반 단방향 해시로 저장

---

### 2️⃣ 일기 관리

- 일기 생성 / 조회 / 수정 / 삭제
- 사용자는 평문으로 일기를 작성
- 서버 저장 시 **AES 기반 암호화** 적용

**데이터 정책**

- 일기 본문: 암호화 저장
- 메타 데이터(작성일 등): 평문 저장

---

### 3️⃣ AI 피드백

- 매일 새벽 03시, Cron 기반 자동 실행
- 누적된 일기 데이터를 분석하여 피드백 생성

**피드백 유형**

- 오늘의 피드백
- 주간 요약
- 월간 인사이트
- 기간 지정 분석

**저장 원칙**

- AI 분석 결과만 저장
- 일기 원문 평문은 저장하지 않음

---

## 🔐 보안 및 개인정보 보호

- 일기 데이터는 **AES-256-GCM** 방식으로 암호화
- 암호화 키는 코드 및 DB에 저장하지 않음
- 복호화는 다음 경우에만 허용
  - 사용자가 자신의 일기를 조회할 때
  - AI 분석 파이프라인 내부 처리 시

---

## ⏰ 기본 시나리오

1. 사용자가 DENZ에 로그인
2. 평문으로 일기 작성
3. 서버에서 암호화 후 DB 저장
4. 새벽 03시 Cron 실행
5. 암호화된 일기 복호화 → AI 분석
6. 피드백 저장
7. 사용자가 설정한 시간에 피드백 제공

---

## 🧠 AI 개인화에 대한 원칙

- DENZ는 사용자 데이터를 AI 모델 학습에 직접 사용하지 않음
- 개인화는 컨텍스트 기반 분석(RAG / 임베딩) 방식으로 구현
- 사용자 데이터는 해당 사용자 경험 개선에만 사용됨
